# 拾光校园 (Shiguang Campus)——大学生校园综合服务平台
## 一、项目基本信息
项目名称：拾光校园 (Shiguang Campus)

项目类型：校园社交 / 环保激励 / Web 应用

## 二、项目背景与定位
### 1. 产品定位
本项目是一款聚焦于高校校园生活的数字化综合平台。致力于连接校园内的活动发现、物品交换、社交互动与环保激励。通过 gamification（游戏化）设计，将学生参与活动、物品循环、社交互动等行为转化为可视化的"光币"资产，打造一个活跃高效的校园生态圈。

### 2. 技术架构策略
混合前端架构：采用 Vue 3 + React 双框架并存的渐进式开发策略。利用 Vue 3 的易用性快速构建基础页面，同时引入 React 生态处理复杂交互，探索前端项目的平滑迁移实践。

现代化 UI 体系：基于 Radix UI 打造简洁交互组件，结合 Tailwind CSS 实现原子化样式管理，确保界面的一致性与美观性。

Serverless 后端：依托 Supabase 构建全栈 BaaS 服务，集成 PostgreSQL 数据库（存储活动/商品/用户关系）、Auth 身份认证（基础用户管理）及 Storage 对象存储（图片资源），保障数据安全与基础功能支撑。

### 3. 数据存储机制
采用"本地优先 (Local-First)"策略。默认利用 LocalStorage 缓存用户数据与应用状态，保障基础浏览体验；支持云端同步，实现数据在多端设备间的基本一致性。

## 三、项目核心意义
### （一）体验层：连接校园，激励参与
活动信息聚合：打破校园活动信息分散的现状，提供统一的活动发布与发现平台，让优质活动能精准触达每一位同学。

环保行为价值化：建立"光币积分"激励体系。系统记录用户的环保贡献（如物品循环、活动参与），转化为可视化的奖励，提升学生的参与积极性和环保意识。

社交互动增强：通过"搭子广场"连接有相似兴趣的学生，促进校园内的社交互动与团队协作。

### （二）普惠性：轻量化与易用性
轻量化访问体验：
免安装 (No-Install)：基于 Web 技术栈，学生通过浏览器即可访问，无需下载安装。

基础加载优化：优先加载核心内容，确保在校园网络环境下也能顺畅使用。

### （三）技术层：渐进式实践与工程价值
渐进式重构范本：在实战中验证 Vue 3 向 React 生态迁移的可行性，为前端项目的技术升级提供参考。

全栈 Serverless 实践：展示了利用 Supabase 快速搭建校园应用系统的能力，利用基础安全策略保护用户数据。

数据驱动的设计：通过用户行为分析，为平台功能优化提供数据支持。

## 四、功能模块详解
### （一）核心业务模块
#### 1. 拾光·活动 (Activities Hub) —— 活动发现中心
活动浏览：展示各类校园活动信息，包括讲座、体育赛事、文艺演出等。

活动报名：支持用户浏览活动详情并进行基础报名操作。

活动分类：按类型筛选活动，便于用户快速找到感兴趣的内容。

#### 2. 益起·流转 (Marketplace) —— 物品交换平台
物品发布：用户可发布闲置物品，支持图片上传和基本信息填写。

物品浏览：提供商品列表和详情页，支持分类筛选。

交易发起：支持用户间的物品交易联系和状态管理。

环保价值：突出物品循环利用的环保意义。

#### 3. 光币中心 (Coins Center) —— 激励积分系统
签到功能：每日签到获得基础光币奖励，连续签到有额外加成。

任务系统：完成发布物品、参与活动、完善资料等任务获得光币。

礼品兑换：光币可兑换预设的礼品奖励。

排行榜：展示用户的碳减排贡献排名，激发参与积极性。

#### 4. 搭子广场 (Buddy Square) —— 社交组队平台
组队需求发布：用户可发布寻找同伴的需求，如一起参与活动、学习等。

需求浏览：其他用户可以浏览并申请加入组队。

基础匹配：基于活动类型进行简单的需求匹配。

### （二）系统界面与交互设计
界面架构：采用清晰的卡片式设计，包含首页导航、活动列表、商品详情、个人中心、光币中心、搭子广场等主要页面。

交互设计：在关键操作节点提供状态反馈和引导提示，提升用户体验。

响应式适配：主要针对移动端优化，确保在手机上良好的使用体验。

### （三）技术保障体系
#### 1. 数据安全与权限
基础权限控制：利用 Supabase RLS 策略，确保用户只能访问自己的数据。

数据保护：保护用户基本信息安全，仅在必要环节显示相关信息。

#### 2. 性能与稳定性
基础性能优化：合理的组件加载和数据请求策略。

稳定性保障：错误处理和基础的离线支持。

#### 3. 可扩展架构
模块化设计：功能模块相对独立，便于后续扩展。

配置化支持：基础的活动类型和商品分类配置能力。
